Call for participation

Overview

Conference metadata are distributed in different sources (e.g. easychair.org, publisher, conference organizer) and encoded in different forms (e.g. HTML, CSV, and even PDF). In the light of offloading tedious routines (e.g. copy and paste, validate and republish) to machine power, RDF-based conference metadata has been collected and published since ISWC2001. This year, we finally achieved a full coverage of ISWC metadata from 2001 to 2013 (see Resources Section. Participants are welcome to enhance the value of the basic ISWC metadata and build cool apps for ISWC2013. Following are some sample ideas:

  • More data, e. g., how to enrich the metadata with conference activity stream?
  • Higher quality, e.g. how to align "Jim Hendler" and "James Hendler"?
  • Social Mash-ups, e.g. how to mashup relevant tweets to an ISWC event?
  • Linked Data Mash-ups, e.g. how to leveage linked data cloud to better find an ISWC participant?
  • Cool conference metadata management apps, e.g. Mobile apps

Demos

Title: Demo1: Generate ISWC 2013 Conference Organizer page (RDFa) from ISWC metadata.
Creator: Li Ding
Homepage: http://lidingpku.github.io/iswc-archive/data/www/iswc-2013-gen_conf_organizer.html
Description: A list of ISWC 2013 conference organizers are maintained a raw CSV file (https://github.com/lidingpku/iswc-archive/blob/master/data/source/iswc-2013-person.csv), each of the first 33 persons is related to role "[ME]" with role type "swc:Chair". A simple query over the CSV data, i.e. (name, organization, country, homepage, role_label), yields an HTML file (http://lidingpku.github.io/iswc-archive/data/www/iswc-2013-gen_conf_organizer.html), including one section for proceedings chair and one section for website maintainer.
Title: Demo2: Fluid Conference metadata Visualization
Creator: Fluid
Homepage: TBA
Description: We are using data from different Semantic Web conferences for our Conference Explorer system (http://conference-explorer.fluidops.net). It is built using our Information Workbench platform and allows exploring, searching, and visualizing various information about major Semantic Web conferences. Currently we are working on a project which aims at using these data to implement a mobile conference assistant with the use of the upcoming Google Glass. For this, we will need to be able to perform inferences over the data using both generic OWL and custom rules, and inconsistencies with the schema ontologies create problems for that: that is the reason why it would be good for us to have the data in a cleaner form than the previous datasets.
